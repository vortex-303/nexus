AI, Together

Stop using AI alone.
Start using it as a team.

Most teams use AI in private — one person prompts ChatGPT, pastes the output, and nobody knows what was asked or why. Nexus makes AI a shared, transparent experience.

The old way
  • Prompt AI alone in a separate tab
  • Copy-paste results into Slack
  • No one sees the prompt or the model
  • Context lost between conversations
  • Every teammate has a different AI setup
  • AI output treated as someone's opinion
The Nexus way
  • @Brain in any channel — everyone sees it
  • The model, the prompt, the reasoning — all visible
  • Brain remembers every conversation across the team
  • Shared agents with shared skills and knowledge
  • One AI teammate that knows the whole workspace
  • AI output attributed, searchable, and accountable

Full transparency

Every AI interaction happens in a channel. The team sees what was asked, which model answered, and what tools were used. No black boxes.

Shared context

Brain accumulates knowledge from every conversation, decision, and document. When anyone asks a question, Brain draws on the whole team's context — not just one person's chat history.

Accountable AI

AI responses are attributed, timestamped, and searchable. Months later, you can trace any AI-generated insight back to the original question and the context it was given.

Tasks & scheduling

Roles & Permissions

9 roles from guest to super admin, 22 granular permissions, and an org chart. Control exactly who sees what, who can invite, and who manages agents.

Get started

Five ways to start.
One brain.

Pick the path that fits your workflow. All five run the same Nexus.

Fastest

Nexus Cloud

Name your workspace, pick a display name, done. No install, no config.

Open Nexus Cloud ~10 seconds

Install Script

One-line installer for Linux servers. Downloads the latest release binary.

$ curl -fsSL https://raw.githubusercontent.com/vortex-303/nexus/main/install.sh | sh
$ nexus serve
~30 seconds

Docker

Single command with Docker Compose. Add SSL with the production config.

$ curl -fsSL https://nexusteams.dev/install-docker.sh | bash
~1 minute

Fly.io

Deploy to Fly.io's global edge network. Your own private instance with persistent volume.

$ fly launch --from https://github.com/vortex-303/nexus
~2 minutes

Build from Source

Clone the repo, build, run. Requires Go 1.25+ and Node 22+.

$ git clone https://github.com/vortex-303/nexus
$ cd nexus && make build && ./nexus serve
~2 minutes
